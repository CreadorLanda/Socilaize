# 🤝 Guia de Contribuição

> Guia para contribuir com o Socialize.

## 🎯 Como Contribuir

### Formas de Contribuir

| Forma | Descrição |
|-------|-----------|
| 🐛 Reportar Bugs | Ajude a encontrar bugs |
| 💡 Sugerir Features | Proponha novas ideias |
| 📝 Documentação | Melhore docs |
| 💻 Código | Implemente features |
| 🎨 Design | Melhore UI/UX |
| 🌍 Tradução | Traduza para outros idiomas |

---

## 🚦 Primeiros Passos

### 1. Fork o Repositório

```bash
git clone https://github.com/YOUR_USERNAME/socialize.git
cd socialize
```

### 2. Configure o Ambiente

Siga o [Guia de Setup](./docs/contrib/setup.md).

### 3. Crie uma Branch

```bash
git checkout -b feature/minha-feature
# ou
git checkout -b fix/meu-bug
```

### 4. Faça suas Mudanças

```bash
# Faça as mudanças necessárias
git add .
git commit -m "feat: minha nova feature"
```

### 5. Abra um Pull Request

```bash
git push origin feature/minha-feature
```

---

## 📋 Commits

### Conventional Commits

```
feat: nova feature
fix: bug fix
docs: documentação
style: formatação
refactor: refatoração
test: testes
chore: manutenção
```

### Exemplos

```
feat: adiciona modo ghost
fix: corrige login em branco
docs: atualiza getting started
refactor: simplifica auth middleware
```

---

## 🔍 Pull Request

### Requisitos

- [ ] Código segue [Style Guide](./docs/contrib/style-guide.md)
- [ ] Testes passando
- [ ] Documentação atualizada
- [ ] Changelog atualizado
- [ ] Sem conflitos com main

### Processo de Review

1. **CI** rodando
2. **Code review** pelos mantenedores
3. **Ajustes** se necessário
4. **Merge** aprovado
